# Файлы состояния

## `posts.json`

- Назначение: хранение постов и отложенных публикаций.
- Читается и пишется функциями `load_posts` / `save_posts`.

## `users.json`

- Назначение: баланс пользователя и счётчик `total_generations`.
- Нормальный формат: JSON-объект вида `{ "<telegram_user_id>": { ... } }`.
- Факт: в текущем workspace файл был найден в устаревшем формате `[]`.
- Админская ветка может создавать запись лениво, если Telegram ID уже известен, но user-record ещё не существует.

## `billing_config.json`

- Назначение: хранение `markup`, то есть стоимости одной AI-генерации.
- Значение по умолчанию: `30.0`.

## `system_prompt.txt`

- Назначение: шаблон промта, который форматируется перед вызовом OpenRouter.
- Меняется из Telegram admin panel.
- Если файл пуст или отсутствует, код подставляет встроенный дефолтный шаблон.

## `vacancy_config.json`

- Назначение: шаблоны вакансий, диапазоны зарплат, offer/requirements, ссылки и каталоги с фото.
